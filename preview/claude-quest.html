How anyone picks their hero

It's one file — no downloading a different script per character. Each person runs one command:

python ~/.claude/statusline-game.py --list
python ~/.claude/statusline-game.py --set pika

That writes a tiny statusline-hero.txt the script reads. Priority: CLAUDE_SL_SPRITE env → config file → default. Adding a new hero = one more entry in the HEROES dict.

How it actually animates

Re-runs on events (each assistant message, debounced 300 ms) + an optional refreshInterval timer — min 1 second. No smooth 60 fps; this ~1 fps bob is the ceiling.

So position carries the meaning (usage); the tick just adds life + the jump. Already live in your config (refreshInterval: 1).

Rollback: point command back to statusline.py.

Zones: green <70% · yellow ≥70% · red ≥90% → JUMP
The terminal render is a touch chunkier (half-block pixels); the sprite art is identical. Restart Claude Code if the new statusline doesn't appear on the next message.
